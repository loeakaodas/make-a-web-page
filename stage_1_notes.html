Basics of the Web

The World Wide Web is a collection of HTML documents that are linked together by URLs and HTTP across the internet. The Web itself was invented in the early 90's at CERN and now comprises over 30 billion pages. You access the web through a browser, which sends a request across the internet using HTTP to a server; a specialized computer designed to store websites. The most popular browsers are Internet Explorer, Chrome, Firefox.

HTML

HTML stands for Hypertext markup language. An HTML document would contain all the text, markup, references, and links to other pages that a browser would interpret to present a complete webpage to an end user.

Markup

HTML markup consists of tags & elements. A tag is something that can change the behavior of it's contents, for example making it bold or italic. It is important to remember to close all your tags or risk creating problems, there are however void tags that don't need to be closed: ex. the img or br tags. There are many different tags that do different things, you can for example link to the Udacity homepage! And many other fun things as well. A complete tag comprises an element.

A good thing to also remember is how whitespace is treated. All tabs, new lines, or multiple spaces get translated into a single space & that is why you need markup to change things up.

Computers are Stupid

Computers are very literal and logical (usually). They will take all your instructions exactly as they were written, no how they might have been intended. So it's important to follow the correct syntax when coding.

You can't remember everything!

So don't even try. Knowing how to find the right answer with a google search or documentation is always a wiser approach anyway.

Inline vs Block Elements

HTML treats certain elements as inline and other as block. What that means is that inline elements will appear on the same line, while

block elements get their own special "box" or block.

Lesson 2 Notes

Structure of HTML documents

A web browser interprets HTML's tree-like structure (DOM-document object model) and presents it in a series of nested boxes, aligned to a grid, and according to the rules that HTML's DOM has established.

Chrome Dev Tools

Chrome's Dev Tools will show you how a page is structured and allow you to manipulate and instantly see the results on the page. They also demonstrate how the different indentation of the HTML document transforms the element into a hierarchy of nested boxes.

Lesson 3 Notes

Adding Style with CSS
  • CSS: cascading style sheets
  • Can have multiple CSS files referenced in an HTML document
  • CSS creates the look & feel of a page
  • Cascading means you keep going down until you find the rule that is most specific to the element
  • Can also think of CSS as search & replace
    • find the class or tag of an element & then apply that rule to it
  • Cascading is an adaptation of the programming principle of inheritance
Reducing complexity with CSS

By utilizing CSS you can reduce the complexity & repetition of having to edit individual sections of HTML in order to change how it looks in the browser. Instead you just apply the desired style with CSS & as a result it can be easily changed whenever necessary without having to edit the HTML extensively or at all.

Validating your code

Once you have your code complete it is important to validate it using the W3C validation tools for HTML & CSS so that you can find mistakes that might make the pages appear differently on different browsers and find out if you are conforming to the HTML5 standard or not in your code.
